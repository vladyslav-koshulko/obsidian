PKCE (Proof Key for Code Exchange, вимовляється "піксі") — це ==розширення протоколу OAuth 2.0, призначене для захисту авторизаційних кодів від перехоплення==, що особливо актуально для мобільних та односторінкових додатків (SPA). Воно гарантує, що лише той додаток, який ініціював запит, може обміняти код на токен доступу. 

**Як працює PKCE (основні кроки):**

1. **Створення секрету:** Клієнтський додаток генерує випадковий рядок (`code_verifier`) і його хеш (`code_challenge`).
2. **Запит авторизації:** Додаток відправляє `code_challenge` на сервер авторизації.
3. **Авторизаційний код:** Сервер повертає код.

- **Обмін токена:** Додаток відправляє код і оригінальний `code_verifier`.
- **Перевірка:** Сервер перевіряє, чи відповідає verifier раніше отриманому challenge. Якщо ні, токен не видається.

**Ключові переваги:**

- **Захист від перехоплення:** Навіть якщо зловмисник вкраде код, він не зможе його використати без `code_verifier`.
- **Універсальність:** Рекомендується для всіх клієнтів OAuth, включаючи веб-додатки.
- **Безпека:** Замінює необхідність використання статики (client secret) у ненадійних середовищах.

PKCE офіційно визначено в [RFC 7636](https://www.rfc-editor.org/rfc/rfc7636.html).
